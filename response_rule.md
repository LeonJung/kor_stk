1. **think.md 갱신** — 작업 중 떠올린 추론, 결정의 근거, 고민, 문제 분석 흐름을 일자시간 헤더(`## YYYY-MM-DD HH:MM KST`)와 함께 누적 기록한다. 단답이 아닌 흐름이 보이도록 1–2문단씩. 과거 작업도 가능한 범위에서 소급 정리.

2. **todo.md 갱신** — 사용자에게 "대안 / 다음 후보 / 선택지 / option" 으로 제시한 모든 항목은 todo.md 에 남긴다. 결정 후 상태 표시:
   - `[x]` 채택/완료
   - `[~]` 보류
   - `[d]` 폐기/리젝트
   결정된 것은 "Decided" 섹션으로 옮겨 시간순 누적.

## 2026-05-08 02:05 KST — 사이즈 관리 + 재확인 빈도 조정

4. **think.md 사이즈 관리** — 매 턴 think.md 전체를 읽으면 컨텍스트가 부풀어 올라 비효율. 다음 임계로 자동 관리:
   - **50KB 초과 또는 ~1500줄 초과**: Read 호출 시 마지막 ~30개 섹션만 읽음 (전체 X).
   - **100KB 초과 또는 30일 이상 누적된 섹션 존재**: 30일 이상 된 섹션을 `think_archive/think_YYYY-MM.md` 로 옮기고 본 파일은 최근만 유지.
   - 분리·정리는 Claude 가 자동 수행 (사용자 별도 지시 없이).

5. **컨텍스트 재확인 빈도 — 4턴에 1번꼴** (rule 3 대체):
   - `.claude/turn_count` 파일에 정수 카운터 유지. 매 새 사용자 명령마다 카운터를 1 증가시켜 다시 저장.
   - 카운터 값 % 4 == 1 (= 1, 5, 9, 13, …번째 명령) 인 턴에서만 `response_rule.md` 와 `think.md` 를 Read 도구로 읽어 컨텍스트 갱신.
   - 그 외 턴에서는 두 파일 Read 생략 — 직전 컨텍스트로 충분하다고 가정.
   - **새 세션의 첫 명령**에서는 카운터 값과 무관하게 두 파일을 무조건 Read (이전 세션 컨텍스트가 비어 있으므로).
   - 사용자가 "rule 확인 / think 봐 / 룰 보고 해" 등 명시 요청 시 즉시 Read.
